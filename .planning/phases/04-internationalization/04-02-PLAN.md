<task type="auto">
  <name>Task 1: Fix dark mode flash on language switch</name>
  <files>src/providers/ThemeProvider.tsx</files>
  <action>
    In ThemeProvider.tsx, change `disableTransitionOnChange={false}` to `disableTransitionOnChange={true}`.

    This tells next-themes to temporarily add a style tag that disables all CSS transitions whenever the theme context re-renders (which happens during locale navigation). The inline disable-transitions script in layout.tsx only runs on initial HTML load, not during client-side navigation, so next-themes must handle it internally.

    Root cause from debug session: React re-renders during router.push() trigger theme provider updates, and the 150ms CSS transitions animate theme colors causing a visible flash.
  </action>
  <verify>
    <automated>cd /Users/emanuelpereyra/gitRepos/own/personal-site && grep -q 'disableTransitionOnChange={true}' src/providers/ThemeProvider.tsx && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>ThemeProvider has disableTransitionOnChange set to true. No CSS transition flash during language switching in dark mode.</done>
</task>

<task type="auto">
  <name>Task 2: Fix scroll position loss on language switch</name>
  <files>src/components/LanguageSwitcher.tsx</files>
  <action>
    In LanguageSwitcher.tsx, modify the `handleLanguageSwitch` function to include the hash fragment in the router.push call.

    Current code (line 18):
    ```
    router.push(pathname, { locale: newLocale });
    ```

    Change to:
    ```
    router.push(`${pathname}${hash}`, { locale: newLocale });
    ```

    This passes the hash fragment (e.g., "#experience") as part of the destination path, so next-intl's router navigates to the correct section in the new locale.

    Also remove the now-unnecessary setTimeout block (lines 20-28) that tried to restore the hash after navigation. With the hash included in router.push, the browser handles scroll restoration natively. The setTimeout with scrollIntoView was a workaround that races with navigation and fails.

    Root cause from debug session: usePathname() returns only the pathname (e.g., "/") without the hash fragment. The hash was captured but never passed to router.push.
  </action>
  <verify>
    <automated>cd /Users/emanuelpereyra/gitRepos/own/personal-site && grep -q 'router.push.*hash.*locale' src/components/LanguageSwitcher.tsx && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>Language switch includes hash fragment in navigation. Switching from /en/#experience to ES navigates to /es/#experience preserving scroll position.</done>
</task>
