**For Portfolio Section:**

**Source:** Existing Skills section (badge spacing), Timeline section (card padding), Contact section (button patterns)

- **Card background:** white (light) / gray-800 (dark) — consistent with professional card styling

- All text meets 4.5:1 contrast ratio minimum

| Empty state heading | Not applicable — no empty state (4 projects hardcoded) |
| Empty state body | Not applicable |

| Destructive confirmation | Not applicable — no destructive actions |

**For Portfolio Section:**
- **Section heading:** "Portfolio" (existing translation key: `portfolio.heading`)
- **Button labels:**
  - `portfolio.viewProject` (EN: "View Project", ES: translation in es.json)
  - `portfolio.viewCode` (EN: "View Code", ES: translation in es.json)
- **Accessibility labels:**
  - `aria-label="View {project.name} live demo"` for demo links
  - `aria-label="View {project.name} source code on GitHub"` for repo links
  - Dynamic aria-label construction ensures screen reader clarity

**Handling Multiple Demo URLs:**
- Projects 1 and 2 (Agoda, Rocket Travel) have `alternateUrl` field with second demo URL
- **Decision:** Display both URLs as separate "View Project" buttons with descriptive aria-labels distinguishing them
- Example: `aria-label="View Agoda white labeling themes - ANA World Hotel"` and `aria-label="View Agoda white labeling themes - Citibank Travel"`
- Buttons displayed horizontally with gap-3 spacing

**Conditional Rendering:**
- If `demoUrl` exists: show "View Project" button with ExternalLink icon
- If `alternateUrl` exists: show second "View Project" button (different aria-label)
- If `repoUrl` exists: show "View Code" button with Code icon
- If no links exist (hypothetical): no action buttons rendered — only title, description, and tech badges

**Source:** Existing translation keys in en.json/es.json (lines 207-211), CONTEXT.md D-14 to D-17, RESEARCH.md Pitfall 6

## Visual Specifications

### Card Design
- **Shadow:** shadow-md base, shadow-lg on hover — Material Design inspired depth (D-09)
- **Border radius:** rounded-lg (8px) — consistent with modern corporate aesthetic (D-10)
- **Transition:** transition-shadow duration-300 — smooth hover effect
- **Padding:** p-6 (24px) — matches Timeline card density
- **Hover behavior:** Slight shadow increase (shadow-md → shadow-lg) indicates interactivity

### Tech Badge Design
- **Style:** Same rounded badge pattern as Skills section (D-11)
- **Classes:** `inline-block rounded-full px-3 py-1.5 text-sm font-medium`
- **Color:** Neutral gray (D-12) — `bg-gray-200 text-gray-700 dark:bg-gray-700 dark:text-gray-300`
- **Spacing:** gap-2 (8px) between badges, flex-wrap for horizontal layout
- **Hover:** transition-transform hover:scale-105 — subtle interactivity (matches Skills badges)

### Button Design
- **Style:** Outline/ghost (D-17) — border with colored text, transparent background
- **Classes:** `inline-flex items-center gap-2 rounded-lg border-2 px-4 py-2 text-sm font-medium transition-colors`
- **View Project (primary):** `border-blue-600 text-blue-600 hover:bg-blue-50 dark:border-blue-400 dark:text-blue-400 dark:hover:bg-blue-950`
- **View Code (secondary):** `border-gray-600 text-gray-600 hover:bg-gray-50 dark:border-gray-400 dark:text-gray-400 dark:hover:bg-gray-900`
- **Icon size:** h-4 w-4 (16px) — proportional to text-sm button label
- **Icon + text gap:** gap-2 (8px) within button

### Responsive Behavior
- **Mobile (< 1024px):**
  - Single column grid (grid-cols-1)
  - Full width cards with 24px gap (gap-6)
  - Buttons may wrap to 2 rows if both demo and code links exist
- **Desktop (≥ 1024px):**
  - 2-column grid (lg:grid-cols-2)
  - 4 projects = 2 rows × 2 columns
  - 32px gap between cards (lg:gap-8)
  - Buttons display in single row with gap-3

**Source:** CONTEXT.md D-08 to D-17, RESEARCH.md Pattern 2 and Pattern 3, existing Skills and Timeline responsive patterns

## Notes for Planner

1. **Multiple demo URLs:** Projects 1 and 2 have `alternateUrl` field. Display both as separate "View Project" buttons with descriptive aria-labels. See Copywriting Contract section for implementation details.

2. **Translation data structure:** Add `portfolio.projects` array to en.json/es.json with 4 project objects. Each object needs: name, description, technologies[] (array of strings), demoUrl? (optional), alternateUrl? (optional), repoUrl? (optional).

3. **Component structure:** Mirror Timeline pattern — ProjectGrid.tsx (container) and ProjectCard.tsx (item). Both are server components (no "use client" needed for static content).

4. **Accessibility:** External links MUST include `target="_blank"` `rel="noopener noreferrer"` and descriptive `aria-label` attributes. Follow Contact section pattern.

5. **Tech badge colors:** Use neutral gray (gray-200/gray-700) to distinguish from Skills section's category colors (blue/green/purple/orange). This prevents visual confusion between sections.

6. **Testing:** Manual visual testing required at 320px (mobile), 768px (tablet), 1024px+ (desktop), and dark mode toggle. No automated test infrastructure exists (deferred to Phase 8).

7. **Performance:** Server components render faster than client components. ProjectGrid/ProjectCard have no interactivity (no onClick, useState, useEffect), so server-side rendering is optimal.
