<task type="auto">
  <name>Task 2: Fix Contact section detection with adjusted IntersectionObserver</name>
  <files>src/components/Header.tsx</files>
  <action>
Per D-02: Contact section is too short for current -20% top / -80% bottom rootMargin threshold, causing it not to be detected.

Current: IntersectionObserver uses `rootMargin: '-20% 0px -80% 0px'`
Problem: Contact section content is shorter than 20% viewport height, so it never intersects.

Fix options analyzed:
1. Reduce top threshold: `-10% 0px -80% 0px` (allows shorter sections to trigger)
2. Add min-height to Contact section (forces it to be tall enough)

Choose option 1 (adjust threshold) — cleaner, doesn't force artificial height.

Implementation:
In Header.tsx, update the IntersectionObserver options:
```typescript
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        setActiveSection(entry.target.id);
        window.history.replaceState(null, '', `#${entry.target.id}`);
      }
    });
  },
  { rootMargin: '-10% 0px -80% 0px' } // Changed from -20% to -10%
);
```

Why -10%: Allows sections as short as 10% viewport height to be detected, while still requiring them to be prominently in view before activating.
