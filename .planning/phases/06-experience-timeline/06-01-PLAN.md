# Phase 6, Plan 1: Experience Timeline Implementation

> Implement vertical timeline component displaying 19+ years of work history with company names, job titles, date ranges, and achievement bullets in responsive single-column layout

---

## Goal (What DONE Looks Like)

Users visiting the portfolio site can:
1. **View complete work history** — 12 positions from May 2007 to Present displayed in reverse chronological order (most recent first)
2. **Read key achievements** — 2-3 impactful achievement bullets per position, extracted from LinkedIn profile
3. **Scan timeline visually** — Vertical connector line with dots creates clear chronological visualization
4. **Experience responsive design** — Single-column vertical layout with enhanced spacing on desktop, maintaining consistency from mobile (320px) to desktop (1440px)
5. **Switch languages seamlessly** — All timeline content (company names, titles, dates, achievements) displays in English or Spanish via translation files

**Success criteria (goal-backward validation):**
- Navigate to /#experience section → timeline renders with 12 entries, intive (April 2025 - Present) at top, Aguas de Santiago (May 2007 - February 2011) at bottom
- Each entry shows: company name (heading), job title, date range in "Month YYYY - Month/Present" format, 2-3 achievement bullets
- Visual elements: blue dots aligned to timeline connector line (gray), content cards extend rightward with proper spacing
- Responsive check: 320px mobile shows vertical timeline with tight spacing, 1440px desktop shows same layout with generous breathing room
- Toggle dark mode → timeline adapts (dots: blue-400, connector: gray-700, text: gray-300)
- Switch to Spanish (/es/) → all content displays in Spanish (company names, titles, dates with Spanish months, achievement bullets translated)

### Task 1.2: Create TimelineItem component with Flexbox layout
**What:** Build TimelineItem component with left marker (dot + line) and right content card
**Why:** Flexbox layout creates classic timeline visual pattern with minimal CSS complexity
**How:**
- Create `src/components/Timeline/TimelineItem.tsx`
- Define props type: `{ position: { company, title, startDate, endDate, achievements }, isLast: boolean }`
- Render `<li>` with `flex gap-4 lg:gap-6` (horizontal Flexbox)
- Left column: timeline marker
  - Outer div: `relative flex flex-col items-center` (vertical Flexbox for dot + line)
  - Dot: `<div className="z-10 h-3 w-3 rounded-full bg-blue-600 dark:bg-blue-400" />`
  - Connector line: `{!isLast && <div className="w-0.5 flex-1 bg-gray-300 dark:bg-gray-700" />}` (hidden on last item)
- Right column: content card
  - Outer div: `flex-1 pb-8 lg:pb-12` (flex-1 takes remaining space, pb for spacing)
  - Company heading: `<h3>` with text-lg font-semibold text-gray-900 dark:text-white
  - Job title: `<p>` with text-base font-medium text-blue-600 dark:text-blue-400
  - Date range: `<p>` with text-sm text-gray-500 dark:text-gray-400, format `{startDate} - {endDate}`
  - Achievements: `<ul className="list-disc space-y-1 pl-5 text-gray-600 dark:text-gray-300">` with `<li>` for each bullet

**Acceptance:**
- [ ] TimelineItem renders with dot on left, content on right (Flexbox layout)
- [ ] Dot is blue-600 (light mode) / blue-400 (dark mode), 12px diameter
- [ ] Connector line shows for all items except last one
- [ ] Content card displays company name (heading), job title, date range, achievement bullets
- [ ] Achievement bullets render as unordered list with disc markers
- [ ] Dark mode classes applied to all text elements

**Files created:** `src/components/Timeline/TimelineItem.tsx`

## Risk Assessment

### Technical Risks

**Risk 1: Translation file JSON syntax errors**
- **Likelihood:** Medium (manual JSON editing prone to syntax errors)
- **Impact:** High (breaks entire site if JSON invalid)
- **Mitigation:** Validate JSON after each translation file edit, run `npm run dev` after Wave 0 checkpoint
- **Fallback:** Use JSON linter (VS Code extension) or online validator before committing

**Risk 2: Timeline connector line extending below last item**
- **Likelihood:** Medium (common CSS pitfall with last-child handling)
- **Impact:** Low (visual defect, no functional impact)
- **Mitigation:** Pass `isLast` prop to TimelineItem, conditionally render connector line with `{!isLast && ...}`
- **Fallback:** Use CSS `:last-child` selector if props approach fails

**Risk 3: Dark mode contrast issues**
- **Likelihood:** Low (following established color patterns from Phase 3/5)
- **Impact:** Medium (accessibility concern, WCAG AA compliance)
- **Mitigation:** Test dark mode explicitly in Task 2.3, adjust gray shades if needed
- **Fallback:** Use lighter gray shades (gray-300 instead of gray-400) for better contrast

**Risk 4: Mobile horizontal overflow**
- **Likelihood:** Low (Flexbox with flex-1 should prevent overflow)
- **Impact:** Medium (poor mobile UX, horizontal scroll)
- **Mitigation:** Test on 320px viewport in Task 2.2, ensure text wrapping and proper padding
- **Fallback:** Reduce left marker width or increase content `flex-shrink` if overflow occurs

**Risk 5: Spanish translation quality**
- **Likelihood:** Low (AI translation generally accurate for professional content)
- **Impact:** Medium (unprofessional if translations awkward)
- **Mitigation:** Review Spanish translations for professional tone, verify job titles use standard terms
- **Fallback:** User can refine translations post-phase if needed (content-only change)
